# Cache Simulator

This project implements a simple set-associative cache simulator in C++. It reads a memory access trace, applies LRU, FIFO, or RANDOM replacement, tracks hits and misses, and reports write-back activity for dirty evictions.

## Features

- Set-associative cache model with configurable cache size, block size, and associativity
- LRU, FIFO, and RANDOM replacement policies
- Read and write hit/miss tracking
- Dirty-line write-back accounting
- Summary statistics after the trace finishes

## Build

From the `Project_cs204` directory, compile with:

```bash
g++  main.cpp cache.cpp policies.cpp -o cache_sim
```

## Run

Launch the program and provide the requested inputs:

```bash
./cache_sim
```

The program prompts for:

1. Cache size in KB
2. Block size in bytes
3. Associativity, as the number of lines per set
4. Replacement policy as an integer:
	- `0` = `lru`
	- `1` = `fifo`
	- `2` = `random`
5. Path to the trace file

Example:

```text
Enter cache size (in KB): 64
Enter block size (in bytes): 4
Enter associativity (number of lines per set): 2
Enter policy (0 = lru, 1 = fifo, 2 = random): 2
Enter trace file path: trace.txt
```

## Trace Format

Each line in the trace file should contain an operation and a hexadecimal address:

```text
R 0x0
W 0x4
R 0x8
```

- `R` means read
- `W` means write
- Addresses are read as hexadecimal values

These trace files are generated by the Pin-tool memtrace2.cpp in the project.

## How It Works

- The cache is split into sets based on the chosen cache size, block size, and associativity.
- On each access, the simulator checks whether the tag is already present in the target set.
- On a hit, it updates the relevant counters and marks the line as dirty on writes.
- On a miss, it selects a line using the configured replacement policy.
- If a dirty line is evicted, the simulator counts a write-back and prints `Write-back occurred`.
- At the end, the program prints total accesses, read/write breakdowns, hit/miss counts, and percentages.

## Project Files

- `main.cpp`: command-line interface and trace processing
- `cache.h`: cache data structures and function declarations
- `cache.cpp`: cache initialization, access handling, and statistics
- `policies.h` / `policies.cpp`: replacement-policy logic
- `trace.txt`, `trace2.txt`, `trace3.txt`: sample traces

## Notes

- The current CLI accepts integer policy codes. Any value other than `0` or `1` falls back to RANDOM behavior in `main.cpp`.
- Addresses are masked into a 256 KB range before indexing, matching the current implementation.
